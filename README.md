## Principes Solides

Les principes SOLID sont cinq directives de conception orientée objet qui visent à amortir l'impact des dépendences et aboutir aux qualités tel que la Robustesse,l'extensibilité et réutilisabilité. Il repose sur cinq principes de conception notament: 

      Le Single Responsability Principle(SRP),ici Une classe doit avoir une seule raison d'être modifiée donc une seule responsabilité est égale  une seule raison d'etre.
  
      Open-Closed Principal(OCP),le code doit être ouvert à l'extension mais fermé à la modification.En résumé,on doit pouvoir ajouter de nouvelles fonctionnalités sans modifier le code qui existe déjà.

      Liskov Substituation Principal(LSP): ici l'idée est pouvoir remplacer une classe mère par une classe fille sans casser le programme,En résumé on doit pouvoir placer les sous classes partout ou figure la classe parent.

       Interface Segregation Principal(ISP): Dans ce principe il faut préférer les petites interfaces ciblées c'est à dire que la dépendance d'une classe à une autre doit etre restreinte à l'interface la plus petite possible.

       Dependency Iversion Principle(DIP) : Ce principe met en evidence le fait que les principes de hauts niveaux(aspect métier) ne doivent pas dépendre des modules de bas niveau(aspect implémentation).

 En conclusion, les principes SOLID sont des guides fondamentaux pour tout développeur souhaitant produire un code de qualité professionnelle.
